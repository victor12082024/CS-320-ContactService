# CS-320-ContactService

•	How can I ensure that my code, program, or software is functional and secure?

To make sure my code is both functional and secure, I focus on testing it the right way from the start. I don’t just check if it works in normal situations, I also test invalid inputs, edge cases, and anything that could break the system. That includes things like null values, length limits, duplicate data, and bad inputs. On the security side, I make sure the system validates everything coming in and doesn’t just trust user input. I also think about how data is handled and make sure nothing sensitive is exposed or easily manipulated. At the end of the day, it comes down to building off the requirements, testing both success and failure cases, and not assuming anything will “just work.”

•	How do I interpret user needs and incorporate them into a program?

I interpret user needs by focusing on what they’re actually trying to do, not just what they say they want. A lot of times users describe features, but the real goal is solving a problem or making something easier. Once I understand that, I turn those needs into clear requirements and build the system around them. From there, everything I design or code ties back to those requirements. That helps make sure the program actually works for the user instead of just being technically correct. If it doesn’t solve their problem or make things easier, then it’s not really a good system.

•	How do I approach designing software?

When I design software, I start with the requirements and break everything down into smaller parts that each handle a specific job. I think about how each part works on its own and how everything connects together as a full system. I also keep things like security, reliability, and future updates in mind so I’m not building something that’s going to fall apart later. For me, it’s not just about making something that runs, it’s about making something that’s organized, easy to understand, and easy to improve over time. That kind of approach makes testing easier and helps avoid problems down the road.
